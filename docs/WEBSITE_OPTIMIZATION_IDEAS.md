# YU YAKANG AUDIO — 网站优化规划（Ideas）

> 状态：规划文档 · 2026-07-05  
> **本文档仅记录后续优化方向，不包含实现任务。**  
> 技术路线冻结：**React + Vite + Express JSON CMS**（无数据库、无复杂权限）

---

|---|---|

## 1. 立即优化

> 可在当前代码基础上推进，主要涉及前台 UI、CMS 字段补全与 JSON 结构扩展，不阻塞部署。

### 1.1 案例详情复盘模板

| 维度 | 说明 |
|---|---|
| **价值** | 将 `background / challenge / solution / result` 等已有字段组织为固定叙事区块（「背景 → 难点 → 方案 → 成果 → 客户反馈」），让演出方/酒吧老板快速扫读专业复盘，提升案例页转化与 SEO 结构化内容密度 |
| **实现难度** | 低 |
| **适合阶段** | 立即 |
| **实现要点** | 扩展 `CaseDetailPage` 区块组件；CMS 无需大改，可选增加 `recapSteps[]` 或使用现有文本字段 + 前端 section 映射；中英文标题走 `i18n` 或 CMS |

### 1.2 证书能力解释

| 维度 | 说明 |
|---|---|
| **价值** | 证书图片 alone 无法传达「这张证代表什么能力」；增加 1–2 行能力说明（如 Smaart 调音认证 → 系统测量与反馈抑制），降低非专业访客理解成本，强化 About / 首页背书区信任感 |
| **实现难度** | 低 |
| **适合阶段** | 立即 |
| **实现要点** | `certificates[]` 增加 `capability.cn/en`（可选 `relatedServices[]` slug 引用）；`CertificateGallery` + Lightbox 展示说明文案 |

### 1.3 设备 / 软件能力墙

| 维度 | 说明 |
|---|---|
| **价值** | 集中展示调音台型号、测量软件（Smaart / EASE）、DAW 与常用插件，回答「你能用什么工具解决我的问题」；对系统工程与混音类客户决策影响大 |
| **实现难度** | 低–中 |
| **适合阶段** | 立即 |
| **实现要点** | 复用 `profile.tools[]`、`profile.skillGroups[]`；About 页新增「能力墙」区块（分类 Tab：现场 / 测量 / 混音）；图标用 Lucide 或静态 logo 图（`public/logos/`），数据仍存 JSON |

### 1.4 首页 Hero 视频进度提示

| 维度 | 说明 |
|---|---|
| **价值** | `caseVideoCarousel` 模式下用户不知还剩几段、总时长；进度条 / 圆点 / 倒计时角标提升首屏完成率，减少误触跳过 |
| **实现难度** | 低 |
| **适合阶段** | 立即 |
| **实现要点** | `HeroVideoCarousel` 根据 `slides.length` 与 `slideDuration` 渲染指示器；纯 CSS + React state，不新增 API；尊重 `prefers-reduced-motion` |

### 1.5 案例列表与详情 SEO 基础补全

| 维度 | 说明 |
|---|---|
| **价值** | 案例页是长尾搜索入口；确保每案 `seo.title/description` 有 fallback（从 title/summary 生成），OG 图默认 `coverUrl` |
| **实现难度** | 低 |
| **适合阶段** | 立即 |
| **实现要点** | `src/lib/seo.js` 工具函数 + 各页 `Helmet` 或 Vite 插件注入 meta；数据已在 `cases[].seo` 预留 |

### 1.6 联系与转化微优化

| 维度 | 说明 |
|---|---|
| **价值** | 强化「第一屏专业感 → 案例证明 → 加微信/预约」链路：Contact 页复制微信号、Booking 成功页二次引导等 |
| **实现难度** | 低 |
| **适合阶段** | 立即 |
| **实现要点** | clipboard API + toast；不新增后端 |

## 2. 后台完成后优化

> 依赖 Admin 各模块可读写、`POST /api/admin/upload` 与简单密码 auth 稳定后再做，避免「只能改 JSON 文件」的运维成本。

### 2.1 后台内容完整度检查

| 维度 | 说明 |
|---|---|
| **价值** | 仪表盘一眼看到哪些案例缺封面、缺英文、缺 SEO、Hero slide 指向 invisible 案例等，减少上线后内容空洞或链接 404 |
| **实现难度** | 中 |
| **适合阶段** | 后台完成后 |
| **实现要点** | `GET /api/admin/content/health` 或在 Dashboard 前端扫描 `site-content.json` 规则集；返回 `{ section, field, severity, message }[]`；规则可配置在 `server/lib/contentHealth.js` |

**建议检查项示例：**

```text
· hero.slides[].caseSlug 是否存在且 visible
· cases[] 必填：title/summary/coverUrl（中英）
· certificates[] imageUrl 文件是否存在
· socialLinks.douyinUrl 是否仍为 self 草稿
· seo.ogImageUrl 是否设置
· 至少 1 条 featured 案例
```

### 2.2 后台媒体使用状态

| 维度 | 说明 |
|---|---|
| **价值** | 上传文件增多后，难以知道某张图是否仍被引用、Hero 视频是否重复占用空间；便于清理 `uploads/` 与 public 静态资源 |
| **实现难度** | 中 |
| **适合阶段** | 后台完成后 |
| **实现要点** | 服务端扫描 CMS JSON 中所有 `*Url` / `images[]` 字段，对比 `public/` 与 `uploads/` 目录；Admin 页展示「已引用 / 孤儿文件 / 缺失文件」三态；**仍用文件系统，不用数据库**

### 2.3 SEO 预览卡片

| 维度 | 说明 |
|---|---|
| **价值** | 编辑全局 SEO 或单案例 SEO 时，实时预览 Google / 微信分享卡片样式，减少「上线才发现标题被截断」 |
| **实现难度** | 中 |
| **适合阶段** | 后台完成后 |
| **实现要点** | Admin SEO / Case 编辑页内嵌预览组件（模拟 SERP snippet + OG card）；字段长度提示（title ≤ 60 字符等）；纯前端 mock，可选调用 `/api/admin/content` 保存前校验 |

### 2.4 Booking 智能引导

| 维度 | 说明 |
|---|---|
| **价值** | 根据用户选择的服务类型、预算区间、活动规模，动态展示推荐案例链接、预计响应时间、需准备的现场信息清单，降低无效咨询、提高表单质量 |
| **实现难度** | 中 |
| **适合阶段** | 后台完成后 |
| **实现要点** | `services[]` 增加 `bookingHints.cn/en`、`relatedCaseSlugs[]`；Booking 表单分步（已有 progress 可扩展）；规则引擎为前端静态映射 + JSON 配置，**无需 AI / 数据库** |

### 2.5 案例编辑与复盘模板联动

| 维度 | 说明 |
|---|---|
| **价值** | Admin Cases 编辑页按复盘模板分 Tab 填字段，与前台 1.1 模板一一对应，编辑体验与展示一致 |
| **实现难度** | 中 |
| **适合阶段** | 后台完成后 |
| **实现要点** | Admin Cases 表单分组 + 必填提示；保存仍 `PUT /api/admin/content/cases` |

### 2.6 预约列表增强

| 维度 | 说明 |
|---|---|
| **价值** | 状态筛选、备注、导出 CSV（从 `bookings.json` 读取），便于手机端快速处理新预约 |
| **实现难度** | 低–中 |
| **适合阶段** | 后台完成后 |
| **实现要点** | 已有 PATCH bookings；补 Admin UI 与 `GET` 筛选 query |

## 3. 第二版创意功能

> 差异化体验与专业工具感，可在 V1 稳定运营、内容充实后再投入。

### 3.1 Before / After 音频对比

| 维度 | 说明 |
|---|---|
| **价值** | 混音 / 现场调音类案例的核心证明手段；A/B 切换或同步播放让访客直观感受处理前后差异，显著高于纯文字 `result` 字段 |
| **实现难度** | 中–高 |
| **适合阶段** | 第二版 |
| **实现要点** | CMS 增加 `audioBeforeUrl` / `audioAfterUrl`（或 `audioPairs[]`）；前端双轨 HTML5 Audio + 简单 UI（同步播放头 / 一键切换）；**第一版仍不引入 wavesurfer**，用原生控件 + 进度条即可；音频文件继续放 `public/audio/` |

### 3.2 声音问题诊断器

| 维度 | 说明 |
|---|---|
| **价值** | 互动问卷式引导（场地类型 → 常见问题：啸叫 / 浑浊 / 音量不够 → 可能原因 + 建议 + 关联案例 + CTA 预约），把官网从「展示站」升级为「轻咨询入口」 |
| **实现难度** | 中–高 |
| **适合阶段** | 第二版 |
| **实现要点** | 决策树 JSON（`src/data/sound-diagnostic.json` 或 CMS `diagnosticFlow`）；纯前端状态机；结果页链到 `/booking?service=…` 与相关案例；**不做真实音频分析或上传录音**（避免隐私与算力成本） |

### 3.3 项目地图

| 维度 | 说明 |
|---|---|
| **价值** | 在地图或城市列表上展示 `cases[].location`，体现地域服务经验（Livehouse / 巡演城市），增强「他在这一带做过活」的信任 |
| **实现难度** | 中 |
| **适合阶段** | 第二版 |
| **实现要点** | CMS 为案例增加 `geo: { lat, lng }` 或仅 `city` 枚举；前台静态 SVG 中国地图高亮或按城市分组列表；**不接入高德/百度 SDK 亦可**（静态图 + 筛选），若接入则注意备案与 Key 管理 |

### 3.4 Hero / 案例视频章节标记

| 维度 | 说明 |
|---|---|
| **价值** | Hero 轮播与案例详情视频支持「跳转到关键片段」（如反馈抑制前后），与现有 `startTime` / `duration` 字段协同 |
| **实现难度** | 中 |
| **适合阶段** | 第二版 |
| **实现要点** | 扩展 `hero.slides[]` 与 `cases[].videoChapters[]`；前台章节列表 UI |

### 3.5 教程 / 经验分享模块深化

| 维度 | 说明 |
|---|---|
| **价值** | 首页已有 `TutorialSection` 占位；第二版可支持短文 + 外链（视频号文章）或静态 MD 页，建立长期 SEO 与专业人设 |
| **实现难度** | 中 |
| **适合阶段** | 第二版 |
| **实现要点** | CMS `tutorials[]` 或 Markdown 文件 + 构建时读取；仍无数据库 |

### 3.6 英文站独立 SEO 与 hreflang

| 维度 | 说明 |
|---|---|
| **价值** | 面向海外巡演 / 国际品牌合作时，英文案例与 meta 需独立优化而非简单语言切换 |
| **实现难度** | 中 |
| **适合阶段** | 第二版 |
| **实现要点** | 现有 `*.cn/en` 字段基础上增加 `link rel="alternate" hreflang`；sitemap 分语言 URL |

## 4. 暂不建议做的功能

> 与当前品牌定位、技术约束或 ROI 不匹配；若未来需求变化可重新评估。

| 功能 | 不建议原因 | 替代方案 |
|---|---|---|
| **用户注册 / 多角色权限系统** | 个人品牌站无协作编辑需求；复杂度高 | 继续「单密码 Admin」；Strapi 迁移时用 Strapi Admin 内置角色即可 |
| **引入 PostgreSQL / MongoDB** | 与 JSON CMS 路线冲突；运维成本上升 | `site-content.json` + `bookings.json`；量大后迁 Strapi（见 DEPLOYMENT_PLAN） |
| **在线实时聊天 / 客服机器人** | 转化主路径是微信 + 表单；机器人易显得不专业 | 微信 QR + Booking 表单 + 可选 sound-diagnostic 引导 |
| **客户端上传录音并 AI 分析** | 隐私、带宽、误判风险；非核心转化 | 第二版「诊断器」仅做问卷引导，不接收音频 |
| **wavesurfer.js 全站波形播放器** | 包体积与移动端性能；V1 需求文档已明确延后 | HTML5 Audio + Before/After 简单对比（第二版） |
| **Lenis 平滑滚动 / 重动效库** | 与「克制、专业」视觉冲突；低端机体验差 | 现有 Motion 微交互 + CSS |
| **Tailwind / shadcn 重构** | 项目已冻结原生 CSS 体系 | 延续 tokens + components.css |
| **自建 CDN / 视频流媒体服务** | 成本与维护远超个人站需求 | 视频放 `public/` 或 Vercel 静态；大文件外链视频号 / 抖音 |
| **案例评论 / 公开评分** |  B2B 专业服务更依赖私信推荐与证书 | `clientFeedback` 字段由本人录入 |
| **完整 WYSIWYG 富文本编辑器** | V1 后台规划为 Textarea + 可选 Markdown | Markdown 预览 + SEO 预览卡片 |
| **小程序 / App** | 官网 + 微信已覆盖目标用户 | 视频号 / 小程序仅作外链入口，不做主产品 |
| **自动混音 / DSP 在线处理** | 超出网站范畴，法律与品质风险 | 案例音频展示 + 预约转化 |

## 5. 推荐优先级（总览）

综合 **转化影响 × 实现成本 × 当前阶段依赖**，建议排期如下：

| 优先级 | 功能 | 阶段 | 难度 | 理由 |
|:---:|---|---|:---:|---|
| **P0** | 案例详情复盘模板 | 立即 | 低 | 案例是核心转化资产，字段已有，只差结构化展示 |
| **P0** | Hero 视频进度提示 | 立即 | 低 | 首屏留存，改动面小 |
| **P0** | 证书能力解释 | 立即 | 低 | 强化背书区，字段扩展轻量 |
| **P1** | 设备 / 软件能力墙 | 立即 | 低–中 | 差异化专业信号，复用 profile 数据 |
| **P1** | 后台内容完整度检查 | 后台完成后 | 中 | 上线前质量闸门，防内容遗漏 |
| **P1** | SEO 预览卡片 | 后台完成后 | 中 | 与 SEO 字段编辑同步，减少运营失误 |
| **P2** | Booking 智能引导 | 后台完成后 | 中 | 提升线索质量，依赖 services CMS 完善 |
| **P2** | 后台媒体使用状态 | 后台完成后 | 中 | 素材增多后才有明显收益 |
| **P2** | 案例编辑与复盘模板联动 | 后台完成后 | 中 | 与 P0 复盘模板配套 |
| **P3** | Before / After 音频对比 | 第二版 | 中–高 | 混音类杀手功能，需准备成对素材 |
| **P3** | 声音问题诊断器 | 第二版 | 中–高 | 互动转化创新，需内容与决策树维护 |
| **P3** | 项目地图 | 第二版 | 中 | 内容足够多后更有说服力 |
| **P4** | 教程模块深化 / hreflang 等 | 第二版 | 中 | 长尾 SEO 与国际化，非上线阻塞项 |

### 建议里程碑

```text
里程碑 A — V1 上线前（立即优化）
  · 复盘模板 + Hero 进度 + 证书说明 + 能力墙 + SEO fallback

里程碑 B — Admin 写能力验收后
  · 内容完整度检查 + SEO 预览 + Booking 引导 + 媒体状态

里程碑 C — 运营 3–6 个月、案例音频素材就绪
  · Before/After + 诊断器 + 项目地图
```
